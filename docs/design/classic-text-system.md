# 古籍演绎系统设计文档

> 创建时间：2026-03-14
> 状态：概念设计阶段

## 一、核心理念

《孙子兵法》等古籍的本质是**博弈与竞争的智慧**，不局限于战争场景。
用 WorldMind 多 Agent 将抽象原则"演"成具体故事，映射到现代生活。

---

## 二、场景泛化

| 孙子原文 | 战争场景 | 商业场景 | 人际场景 | 职场场景 |
|---------|---------|---------|---------|---------|
| "不战而屈人之兵" | 外交施压 | 收购竞对 | 让对方主动让步 | 用数据说服老板 |
| "知己知彼" | 侦察敌情 | 竞品分析 | 了解对方诉求 | 摸清领导风格 |
| "避实击虚" | 绕开主力 | 差异化竞争 | 避开矛盾焦点 | 找领导心情好时谈 |
| "以逸待劳" | 防守消耗 | 后发制人 | 不急于表态 | 等项目完成再谈薪资 |
| "兵贵神速" | 闪电战 | 快速占领市场 | 先发制人表态 | 抢先汇报占据主动 |

**设计**：用户选择场景类型，同一条原则生成不同领域的演绎故事。

## 三、多模态表达方式

### 3.1 文字剧本（基础）
- 结构化对话 + 旁白解读
- 3-5 幕，每幕对应一个策略阶段

### 3.2 语音演绎（Phase 2）
- 每个角色分配不同 TTS 音色（ElevenLabs / Azure）
- 输出 MP3，像"广播剧"
- 适合通勤/睡前场景

### 3.3 漫画分镜（Phase 3）
- 关键节点 → 图片生成（DALL-E / Midjourney）
- 对话气泡叠加
- 输出长图 / PDF 条漫

### 3.4 视频动画（Phase 4）
- 剧本 + 语音 → 虚拟人口型动画（D-ID / HeyGen / Runway）
- 背景场景渲染
- 像"动画短片"

### 3.5 互动模拟（Phase 5 — 最高价值）
- 用户扮演其中一个角色
- 选择不同策略，触发不同剧情分支
- 类似"文字冒险游戏"
- 学习效果最深

## 四、技术架构

```
┌─────────────────────────────────────────────────────────┐
│                    古籍演绎引擎                          │
├─────────────────────────────────────────────────────────┤
│  输入层                                                  │
│  ├─ 古籍原文（某篇原文 + 白话注释）                      │
│  ├─ 场景类型（战争 / 商业 / 人际 / 职场）                │
│  └─ 输出模式（剧本 / 语音 / 漫画 / 视频 / 互动）         │
├─────────────────────────────────────────────────────────┤
│  Agent 层（WorldMind 多 Agent）                          │
│  ├─ 导演 Agent：理解原文，设计剧情结构与冲突点           │
│  ├─ 角色 Agent × N：扮演具体人物，有性格/目标/记忆       │
│  ├─ 旁白 Agent：解读行为背后的兵法原理                  │
│  └─ 评论 Agent：事后复盘，点出关键决策                  │
├─────────────────────────────────────────────────────────┤
│  渲染层（多模态输出）                                    │
│  ├─ 剧本：Markdown / JSON 结构化输出                    │
│  ├─ 语音：角色分音色 TTS → MP3                         │
│  ├─ 漫画：分镜脚本 → 图片序列 → 长图                   │
│  ├─ 视频：图片 + 语音 → 动画合成                       │
│  └─ 互动：状态机 + 用户选择 → 分支剧情                 │
└─────────────────────────────────────────────────────────┘
```

## 五、MVP 路径

| 阶段 | 内容 | 预估时间 | 产出 |
|------|------|---------|------|
| Phase 1 | 剧本生成 | 1 周 | 输入原文+场景 → 3-5 幕对话剧本 |
| Phase 2 | 语音演绎 | +3 天 | 剧本 → 多音色 MP3 |
| Phase 3 | 漫画分镜 | +1 周 | 关键场景 → 图片序列 |
| Phase 4 | 视频动画 | +2 周 | 图片 + 语音 → 动画 |
| Phase 5 | 互动模式 | +2 周 | 用户扮演角色，选择影响剧情 |

## 六、Demo 示例

**输入**
- 原文：《孙子兵法·谋攻》"不战而屈人之兵，善之善者也"
- 场景：职场 — 说服老板批预算

**输出剧本**

```
【背景】
小王想申请 50 万做 AI 项目，老板一向抠门。

【第一幕：知己知彼】
小王（内心）：直接要钱肯定被拒。先摸清老板在意什么……
（调研：老板最近被董事会施压降本增效）

【第二幕：避实击虚】
小王找到老板：
"张总，我发现一个方案，能帮咱们部门省 200 万人力成本。"
（不提要钱，只说省钱）

【第三幕：以利动之】
张总（感兴趣）："怎么省？"
小王："用 AI 自动化处理报表，现在 3 个人干的活，机器 1 小时搞定。"
张总："要投入多少？"
小王："50 万，半年回本，之后每年净省 150 万。"

【第四幕：不战而胜】
张总："这个可以，你写个方案我看看。"
（预算拿到，全程无正面冲突）

【旁白解读】
"不战而屈人之兵"不是逃避，是找到双赢切入点。
小王没有说服老板"给我钱"，而是让老板觉得"这钱花得值"。
```

## 八、交互优化方向

- **场景选择器**：下拉选择场景类型，支持自定义场景描述
- **角色定制**：可替换角色名/性格/背景（如把"小王"换成自己）
- **难度模式**：简单（直接看答案）/ 挑战（先做选择再看结果）
- **分享功能**：生成海报 / 分享链接
- **收藏/回顾**：保存喜欢的演绎故事
